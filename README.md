# РЦТО · CRM путёвок и туров (демо)

Интерактивная демо-CRM для **АНО «Республиканский Центр Туризма и Отдыха УР»** —
альтернатива Битрикс24 под собственный бизнес-процесс: лагеря, санатории, школьные туры.

## Что внутри

4 воронки (как на схемах процесса):

1. **Лиды** — все входящие (ВК, сайт, звонки, педагоги, с улицы) → квалификация.
2. **Физ лица** — продажа путёвок родителям: договор, оплата, заезд, перерасчёт.
3. **ЮЛ · РЦТО** — закупка путёвок санаториев организациями (предоплата/постоплата).
4. **ТО · Туры** — туроператорское направление: расчёт, КП, бронь, акты.

Возможности демо:

- Канбан с **drag-and-drop** сделок между стадиями;
- карточка сделки со всеми полями из ТЗ, оплатой и историей;
- подсказки на стадиях (требования из схемы процесса) — наведите на «i»;
- демо-кнопки **«Позвонить»** (телефония) и **«Выгрузить в 1С»**;
- блок «Интеграции» — как подключаются телефония, 1С, WhatsApp, эквайринг.

## Запуск локально

```bash
npm install
npm run dev
```

## Сборка и публикация на GitHub Pages

```bash
npm run build      # сборка в dist/
npm run deploy     # публикация в ветку gh-pages (нужен git-remote)
```

`vite.config.js` использует `base: './'`, поэтому сборка работает с любого
под-пути GitHub Pages без доп. настройки.

## Боевое внедрение

Демо — статичная витрина (без сервера). Для рабочей версии приложение
разворачивается на облачном сервере (VPS), где подключаются:

- **Телефония** — виртуальная АТС по API (Mango, UIS, Telfin, Zadarma, Sipuni);
- **1С** — двусторонний обмен (HTTP-сервисы / OData / EnterpriseData);
- **WhatsApp, эквайринг, формы сайта и ВК** — через их API.

Стек: React + Vite + Tailwind CSS.
