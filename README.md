# РЦТО · CRM путёвок и туров (демо)

Интерактивная демо-CRM для **АНО «Республиканский Центр Туризма и Отдыха УР»** —
альтернатива Битрикс24 под собственный бизнес-процесс: лагеря, санатории, школьные туры.
Всё работает в браузере (client-side), данные сохраняются в `localStorage`.

## Что внутри

**4 воронки** (как на схемах процесса), канбан с drag-and-drop:

1. **Лиды** — все входящие (ВК, сайт, звонки, педагоги, с улицы) → квалификация.
2. **Физ лица** — продажа путёвок родителям.
3. **ЮЛ · РЦТО** — закупка путёвок санаториев организациями.
4. **ТО · Туры** — туроператорское направление.

**Смарт-процессная карточка сделки** (сделка — корневой элемент):
- внутри сделки — **участники** (родители / плательщики), у каждого свой договор и
  статус подписания;
- у участника — **контакты** (мама + папа с разными телефонами) и **бенефициары**
  (дети / сотрудники / туристы) с полным набором полей (ФИО, дата рождения,
  № свидетельства, гражданство, лагерь, смена, билет);
- суммы сделки считаются автоматически из участников.

**Редактирование** — все поля карточек и реестров редактируются инлайн.

**Реестры:** Контакты, Компании, Лагеря и смены (с загрузкой смен), Документы.

**Генератор Word-документов** (раздел «Шаблоны»):
- загрузите свой `.docx` с тегами `{поле}` и циклом `{#дети}…{/дети}`;
- система находит теги, вы сопоставляете каждый с полем сущности;
- из карточки сделки/участника формируется заполненный `.docx` (скачивается),
  запись попадает в реестр Документов со статусом подписания;
- кнопка «Пример шаблона» отдаёт готовый образец договора.

**Демо-интеграции:** кнопки «Позвонить» (телефония) и «Выгрузить в 1С» в карточке +
раздел «Интеграции».

## Запуск локально

```bash
npm install
npm run dev
```

## Сборка и публикация на GitHub Pages

```bash
npm run build      # сборка в dist/
npm run deploy     # публикация в ветку gh-pages
```

`vite.config.js` использует `base: './'` — сборка работает с любого под-пути.

## Боевое внедрение

Демо хранит данные в браузере (`localStorage`, кнопка «Сбросить демо» в сайдбаре).
Для рабочей версии приложение разворачивается на облачном сервере (VPS):

- **Серверная БД** вместо localStorage, авторизация и права;
- **Телефония** — виртуальная АТС по API (Mango, UIS, Telfin, Zadarma, Sipuni);
- **1С** — двусторонний обмен (HTTP-сервисы / OData / EnterpriseData);
- **WhatsApp, эквайринг, формы сайта и ВК** — через их API.

Стек: React + Vite + Tailwind CSS. Word-генерация: docxtemplater (полностью в браузере).
